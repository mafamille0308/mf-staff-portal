## 1. 目的
- 画面全体の一貫性を高め、操作迷いを減らす。
- 色、配置、ボタン名称、導線を統一し、業務操作を速くする。
- 既存ロジックを壊さず、見た目と操作性を段階的に改善する。

## 2. 現状課題（実装観点）
- 共通CSSはあるが、ページHTML内のインライン `style` が多い。
  - `frontend/js/pages` 全体で `style="..."` が約 494 箇所
  - `settings.js` 単体で約 67 箇所
- ボタン文言の粒度が画面ごとに不統一（「更新」「保存」「実行」「反映」など）。
- 設定画面は情報量が多く、優先操作（プロフィール・通知再送）の視認性が低い。
- 状態表示（badge/toast/エラー文）がページごとに微妙に異なる。

## 3. 改修方針
1. デザイン原則を先に固定する
2. 共通化できる部分を先に共通化する
3. 画面ごとの修正は「導線の重要度順」で行う
4. 1リリース単位で小さく出し、回帰確認を毎回実施する

## 4. デザイン原則（統一ルール）
- 色:
  - 主操作: `btn`（1画面で1系統）
  - 副操作: `btn-ghost`
  - 危険操作: `btn-danger` のみ
  - 成功/注意/失敗は `badge` と `toast` の配色を共通化
- 配置:
  - `section > h1/h2 > 説明 > フォーム > 操作ボタン > 結果表示` の順を基本に統一
  - 主要ボタンはフォーム直下、補助ボタンは右または下へ分離
- 文言:
  - 保存系: `保存`
  - 再取得系: `再読み込み`
  - 実行系: `実行`
  - 再送系: `再送`
  - 破壊操作: `削除` / `無効化` を明示
- 導線:
  - 「よく使う操作」を上段へ寄せる
  - 「管理者専用」は明示バッジを付ける
  - 失敗時は次アクション（何を確認するか）を結果表示に必ず出す

## 5. 実装フェーズ

### Phase U1（短期 / 低リスク）
- 対象: `settings`, `customers_list`, `visits_list`
- 内容:
  - ボタン名称と見出し文言の統一
  - 主要導線の並び順調整（プロフィール最上位・通知再送を上段維持）
  - 結果表示カード（成功/失敗/注意）のテンプレート統一
- 受け入れ条件:
  - 主要操作のボタン文言が統一ルールに一致
  - 操作後のフィードバックが全画面で同じ形式

### Phase U2（中期 / 共通化）
- 対象: `styles.css`, `ui.js`, 各ページ内HTML
- 内容:
  - インライン `style` を共通クラスへ移行（優先: 設定画面）
  - レイアウトユーティリティ追加（例: `stack-8`, `stack-12`, `row-between`, `actions-primary`）
  - 共通カード部品を `ui.js` ヘルパー化（結果カード、エラーガイド、hint-row）
- 受け入れ条件:
  - `settings.js` のインライン style を 67 -> 20 以下に削減
  - 回帰テストで表示崩れなし（主要3画面）

### Phase U3（仕上げ / 一貫性）
- 対象: 全ページ
- 内容:
  - カラーコントラスト最終調整（文字/境界線/disabled）
  - バッジ体系の統一（状態色を定義済みクラスへ寄せる）
  - モーダル/トースト文言のトーン統一（短文 + 次アクション）
- 受け入れ条件:
  - 同種操作でUI表現が揃っている
  - 管理者運用手順で迷う箇所がない

## 6. 画面別の具体修正対象

### 設定
- アコーディオン見出しを目的ベースに再命名
  - 例: `通知` -> `通知再送`
- 各ブロックの説明文を短文化
- 主ボタンと補助ボタンの位置固定

### 顧客
- 顧客詳細のセクション順を「基本情報 -> 連絡先 -> 鍵情報 -> 料金/備考」に統一
- 面談導線のボタン表現を統一（`面談フォームへ` 等）

### 予約
- 予約カードの主情報（日時/顧客/担当/請求状態）を上段へ固定
- 操作ボタンの順序を統一（`保存` -> `請求` -> `キャンセル` -> `再有効化`）

## 7. 回帰確認（各フェーズ共通）
- 画面崩れ:
  - `settings`, `customers`, `visits`, `invoices`
- 操作:
  - 保存/更新/再送/キャンセル/再有効化
- 権限:
  - admin / staff で表示差分が仕様どおり

## 8. 実施順（推奨）

2. Phase U2 で `settings.js` から共通化開始
3. Phase U3 を横断で実施して仕上げ
